# CS 370 Portfolio Artifact Reflection

For this course, I submitted my Jupyter Notebook for the pirate intelligent agent project. This project focused on using reinforcement learning and neural networks to solve a pathfinding problem in a game environment. The goal was to train a pirate non-player character to move through a maze
and reach the treasure. I was given starter code that included the game environment, the maze structure, and support files for storing game experiences. I was also given a notebook with sections that needed to be completed. The main code I created myself was the Q-training algorithm. In that
section, I implemented the training loop, handled exploration and exploitation, stored episodes in replay memory, trained the neural network from replay data, tracked win history, and updated the target network during training.

This project helped me connect several course topics into one assignment. Earlier in the course, I worked with neural networks, reinforcement learning, and Q-learning as separate ideas. In this project, those ideas came together in a more applied way. Instead of only reading about 
intelligent systems, I had to build one and see how training choices affected the outcome. It also showed me that AI systems do not just work because the concept is good on paper. Small details in the algorithm, rewards, and training process can have a big effect on whether the agent 
succeeds or fails.

Computer scientists solve problems by designing systems that can process information, make decisions, and perform useful tasks. That matters because computer science affects many parts of daily life, including communication, business, security, entertainment, and automation. In this project, 
the larger field of computer science showed up through problem solving, algorithm design, debugging, and thinking about how a machine can be trained to make better decisions over time. This matters because intelligent systems are becoming more common, and computer scientists help shape how 
those systems are built and used.

When I approach a problem as a computer scientist, I try to break it into smaller parts and understand how each part works before trying to solve the whole thing. In this project, that meant looking at the environment, the available actions, the reward structure, the neural network, and the 
training loop separately before putting them together. I also had to test different ideas, review errors, and make adjustments when the first version did not work the way I wanted. That process reminded me that problem solving in computer science is often iterative and requires patience, 
testing, and revision.

My ethical responsibilities to the end user and the organization include building systems that are reliable, understandable, and safe to use. Even in a game project, those ideas still matter. A developer should think about whether the system behaves as expected, whether it can be maintained, 
and whether it reflects sound design choices. More broadly, computer scientists have a responsibility to consider how algorithms affect people, how data is used, and whether a system creates unfair or harmful outcomes. This course reinforced that AI is not only about making something work. It 
is also about making sure it works in a responsible way.
